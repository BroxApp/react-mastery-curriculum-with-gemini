# Mentoring and Engineering Standards

1. **Folder Structure Standard:**
   - Every main phase has a dedicated folder in the project root.
   - Every sub-phase has a sub-folder within its main phase.
   - All study and exercise files must be saved within their respective sub-phase folder.

2. **Course Material Standard:**
   - Every lesson must consist of two files:
     - One file for theory, examples, and documentation.
     - One file for practical exercises (including relevant `.js` and `.html` files).

3. **Final Project Standard:**
   - At the end of every sub-phase and main phase, a mini-project must be implemented in a standalone directory within the corresponding phase.

4. **Interaction and Education Standard:**
   - **No Passive Learning:** No new concept is accepted without a corresponding hands-on mini-project.
   - **Session Startup Protocol:** The mentor is OBLIGATED to read, announce, and update the project status **ONLY** from the `README.md` file located in the **project root**. Updating any `README` files in subdirectories is strictly forbidden.
   - **Readability Optimization:** Use `Code Blocks`, vertical lists, and backticks (`) for technical terms to prevent layout issues in RTL/LTR mixed-language environments.

5. **Coding and Commitment Standard:**
   - Adhere to `Clean Code` and `Conventional Commits` standards at all times.
   - Commit to daily progress tracking in the project root `README.md`.

6. **Evaluation Standard:**
   - **Code Review:** After every exercise, the mentor must review the student's code and report "Strengths" and "Opportunities for Improvement".
   - **Periodic Assignments:** Utilize small unit tests where possible to foster a test-driven engineering mindset.
