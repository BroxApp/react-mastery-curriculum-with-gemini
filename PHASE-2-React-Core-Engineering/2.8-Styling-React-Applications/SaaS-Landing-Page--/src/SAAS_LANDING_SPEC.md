🚀 SaaS Landing Page UI Specification
🎯 Objective
Developing a high-converting, modern, and fully responsive SaaS Landing Page using React, Tailwind CSS, and Clean Component Architecture.

🛠 Features Requirements
Responsive Navbar:
Logo, Navigation Links, and a CTA button ("Get Started").
Mobile-friendly: Hamburger menu or vertical list on mobile.
Hero Section:
Eye-catching headline, subtitle, two action buttons, and a mock product image.
Stacked vertically on mobile, side-by-side on desktop (flex flex-col lg:flex-row).
Features Grid:
3 or 4 cards showcasing app features with icons.
Must use CSS Grid/Flexbox: 1 column on mobile, 2 on tablet, 3/4 on desktop.
Pricing Section (Responsive Cards):
3 pricing tiers (Free, Pro, Enterprise).
Highlight the "Pro" card as the most popular (visually distinct).
Footer:
Clean footer with social links and copyright.
🏗️ Technical Constraints
Library: Tailwind CSS (Strictly utility-first).
CSS: No custom standard CSS unless absolutely necessary (for animations, etc.).
Responsive approach: Mobile-First Design.
📅 Milestones
Day 1: Project initialization, Tailwind configuration, and Navbar + Hero Section (Mobile & Desktop).
Day 2: Feature Cards Grid & Pricing Cards (Using Responsive breakpoints).
Day 3: Footer, animations, final polish, and code review.

برای راه‌اندازی پروژه، می‌توانی از همان پروژه ری‌اکتی که در فاز ۲.۸ داری استفاده کنی. مطمئن شو که Tailwind CSS را به درستی طبق مستنداتش در پروژه نصب و پیکربندی کرده‌ای.
از کلاس‌های grid-cols-1 md:grid-cols-2 lg:grid-cols-3 برای بخش ویژگی‌ها استفاده کن.

# 🚀 پروژه: طراحی رابط کاربری SaaS Landing Page

## 🎯 هدف

ساخت یک **لندینگ پیج مدرن، واکنش‌گرا (Responsive)** و با نرخ تبدیل بالا برای یک سرویس SaaS.  
این پروژه با استفاده از **React**، **Tailwind CSS** و معماری کامپوننتی تمیز (Clean Component Architecture) پیاده‌سازی می‌شود.

---

## 🛠 ویژگی‌ها و بخش‌های مورد نیاز

### 1. نوار ناوبری (Navbar)

- شامل لوگو، لینک‌های ناوبری و یک دکمه CTA با متن **"Get Started"**.
- در حالت موبایل باید به‌صورت منوی همبرگری یا لیست عمودی نمایش داده شود.

### 2. بخش هیرو (Hero Section)

- دارای تیتر اصلی جذاب، زیرتیتر توضیحی، دو دکمه‌ی اکشن و یک تصویر ماکاپ محصول.
- در موبایل به‌صورت **ستونی (flex-col)** و در دسکتاپ به‌صورت **سطر افقی (lg:flex-row)** نمایش داده شود.

### 3. بخش ویژگی‌ها (Features Grid)

- شامل ۳ یا ۴ کارت برای نمایش قابلیت‌های اپلیکیشن همراه با آیکون.
- استفاده از **CSS Grid/Flexbox**:
  - موبایل: ۱ ستون
  - تبلت: ۲ ستون
  - دسکتاپ: ۳ یا ۴ ستون
- کلاس پیشنهادی: `grid-cols-1 md:grid-cols-2 lg:grid-cols-3`

### 4. بخش قیمت‌گذاری (Pricing Section)

- شامل ۳ پلن: **Free، Pro، Enterprise**.
- پلن **Pro** باید به‌صورت بصری متمایز و برجسته نمایش داده شود (به‌عنوان پلن محبوب).

### 5. فوتر (Footer)

- فوتر ساده و تمیز شامل لینک‌های شبکه‌های اجتماعی و متن کپی‌رایت.

---

## 🏗 محدودیت‌های فنی

- استفاده از **Tailwind CSS** به‌صورت کامل (Utility-first).
- عدم استفاده از CSS سفارشی مگر در موارد خاص (مثل انیمیشن‌ها).
- طراحی واکنش‌گرا با رویکرد **Mobile-First**.

---

## 📅 مایلستون‌ها

- **روز ۱**: راه‌اندازی پروژه، پیکربندی Tailwind، ساخت Navbar و Hero Section (نسخه موبایل و دسکتاپ).
- **روز ۲**: پیاده‌سازی بخش ویژگی‌ها (Feature Cards Grid) و بخش قیمت‌گذاری (Pricing Cards).
- **روز ۳**: طراحی فوتر، اضافه کردن انیمیشن‌ها، پولیش نهایی و بازبینی کد.

---

## 💡 راهنمایی منتور

- می‌توانی از همان پروژه‌ی ری‌اکتی که در فاز ۲.۸ داری استفاده کنی.
- مطمئن شو که Tailwind CSS را طبق مستندات رسمی نصب و پیکربندی کرده‌ای.
- برای بخش ویژگی‌ها از کلاس‌های **`grid-cols-1 md:grid-cols-2 lg:grid-cols-3`** استفاده کن.
- پروژه را شروع کن و هر زمان اولین بخش (مثلاً Navbar یا Hero Section) را آماده کردی، کد را ارسال کن تا بررسی شود.
